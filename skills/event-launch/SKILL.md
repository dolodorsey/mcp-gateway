# Event Launch Engine

End-to-end event planning, production, marketing, and execution for all Kollective event properties. Every event follows the same disciplined production framework — adapted for scale, market, and brand.

## Core Principle

An event is not a party. An event is a product launch that happens live. It requires the same rigor as launching a brand: strategy, production, marketing, sales, operations, and post-mortem. The Event Launch Engine ensures nothing is improvised that should be planned.

## Flyer Generation Framework

For each event, the flyer package includes:
1. **Primary flyer** (1080×1350 for IG feed)
2. **Story flyer** (1080×1920 for IG/FB stories)
3. **Square variant** (1080×1080 for cross-platform)
4. **Print variant** (if applicable — 24×36" at 300dpi)

### Flyer Must Include:
- Event brand name (prominent)
- Date + day of week
- Time (doors + start)
- City
- Venue name (or "Location upon RSVP")
- Ticket link or RSVP method
- Brand logo
- Social handles
- Age requirement
- Dress code (if applicable)
- Sponsor logos (if applicable)
- Tagline or concept description (1 line)

### Flyer Must NOT:
- Overcrowd with text
- Bury the date or location
- Use off-brand colors or typography
- Miss the CTA
- Look generic or template-based

## Output Modes

### MODE 1: Full Event Plan
Complete event from concept to post-event:
1. Event brief (filled template)
2. Production timeline (T-60 to T+7)
3. Budget framework
4. Marketing plan with content calendar
5. Vendor requirements
6. Staffing plan
7. Run-of-show
8. Post-event workflow

### MODE 2: Event Marketing Campaign
Marketing plan for a specific event:
1. 7-post social campaign
2. 3-email sequence
3. Flyer brief (for creative-production)
4. Influencer outreach plan
5. Paid ad strategy (if budget)
6. PR push plan

### MODE 3: Event Brief Generation
Quick event brief:
1. Filled template with all fields
2. Missing info flagged
3. Budget estimate
4. Vendor needs list

### MODE 4: Run-of-Show
Detailed minute-by-minute event plan:
1. Load-in timeline
2. Setup milestones
3. Doors open sequence
4. Performance/programming schedule
5. Key transitions
6. Wind-down / load-out

### MODE 5: Post-Event Report
Event results and learnings:
1. Attendance vs. target
2. Revenue breakdown
3. Sponsor activation results
4. Content captured
5. What worked / what didn't
6. Vendor performance
7. Recommendations for next event

### MODE 6: Event Calendar View
Multi-event calendar management:
1. All upcoming events with status
2. Production milestone tracking
3. Resource conflicts identified
4. City-level calendar view
5. Team workload assessment
